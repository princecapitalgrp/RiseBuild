Why date-keyed documents?

Streak calculation

Check consecutive date docs exist. No aggregation queries, no counting — just key lookups.

History queries

Last 7 days = where date >= 7-days-ago. One query, O(n) not O(collection).

Idempotent writes

setDoc on a known key = safe re-runs. No duplicate check-ins possible.

Cross-collection joins

Checkin + protocol + reflection for a given day share the same key. Trivial to join.

Security Rules (single rule, total isolation)

rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /users/{uid}/{document=**} {
      allow read, write: if
        request.auth != null &&
        request.auth.uid == uid;
    }
  }
}

One rule covers profile, all subcollections, and memory. No exceptions, no public reads.
